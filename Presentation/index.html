Refactoring
  1. The refacturing began with the separation of the Main program from the game Engine.
  2. Then all methods were extracted.
  3. The project, properties and methods were given names that made sense.
  4. Folders were created and named properly.
Soon the first design pattern was implemented.
  • It is used to handle the execution of all the different commands that the user can enter.
bugs & new features

It was all going smoothly when suddenly...

  • We encountered a bug!
  • The game did not know if the user tried the same letter again and again.

  • We fixed the bug...

  • And implemented a feature that tracks letters used!
the memento pattern
  • Another great feature that was added is the memento design pattern.
  • With it you can save a particular game state and load it later!
  • It saves the current game context to an outer XML file
  • Don't use it to cheat though...
the final feature
  • Last but not least this super awesome and amazing feature was implemented.
  • It saves the final scoreboard to an external txt file.
  • It also loads the scoreboard from it.
  • This is achieved via two very useful methods - Read and Write.
  • Please don't use this to cheat either!
